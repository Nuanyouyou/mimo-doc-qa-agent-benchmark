# Round 1 Results

## 测试任务

基于一份中文产品需求文档，完成摘要、关键信息提取和文档问答。

## 测试输入

测试文档：`sample-docs/product-requirement.md`

测试 Prompt：`prompts/doc-qa-agent.md`

## 初步结果

| 评测项 | 结果 |
|---|---|
| 文档摘要 | 可以生成 100-150 字摘要 |
| 目标用户提取 | 能识别客服团队、电商运营、售后人员等用户 |
| 核心功能提取 | 能提取文档导入、摘要生成、问答、人工确认等功能 |
| 约束条件识别 | 能识别“必须基于文档回答”和“不编造信息” |
| 结构化输出 | 可输出 JSON 格式 |
| 问答表现 | 对文档内问题回答较准确 |
| 潜在问题 | 对文档外问题需要更强拒答约束 |

## 问题与修复记录

### Issue 1: 回答可能超出文档范围

观察：如果用户问题没有明确答案，模型可能会根据常识补充内容。

修复：在 Prompt 中增加“所有回答必须基于文档内容，不要编造文档中不存在的信息”。

### Issue 2: 摘要过长

观察：初始摘要容易超过 150 字。

修复：在任务要求中明确限制“150 字以内”。

### Issue 3: 输出格式不稳定

观察：结构化输出有时不是严格 JSON。

修复：在 Prompt 中增加“最后输出 JSON 格式结果”的要求。
